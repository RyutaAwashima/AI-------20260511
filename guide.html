Aria — Story Generator

Aria の使い方

キーワードを選んでAIにあらすじと世界観を生成してもらうアプリ

1

名前を入力する

ページ上部の「あなたの名前」欄に自分の名前を入力します。この名前はGoogleスプレッドシートに記録されます（本名でなくニックネームでもOK）。

2

キーワードを選ぶ

ジャンル・舞台・テーマ・主人公・雰囲気のカテゴリから、気になるキーワードをクリックして選択します。選んだキーワードがハイライトされます。

  • 各カテゴリで1つ以上選ぶとAIの出力が安定します
  • 複数選択OK。組み合わせが物語の個性になります
  • 迷ったら「おまかせ」ボタンでランダム選択できます
3

「あらすじを生成する」をタップ

ボタンを押すとGemini AIがあらすじ・登場人物・世界観を生成します。生成には10〜30秒かかることがあります。

⏱

スピナーが回っている間は画面を閉じないでください。途中で閉じると生成結果が失われます。

4

結果を確認・編集する

あらすじ・登場人物・世界観の3ブロックで結果が表示されます。内容が気に入らなければ「もう一度生成」ボタンで再生成できます。テキストエリアを直接編集することも可能です。

5

Googleスプレッドシートに保存する

「スプレッドシートに保存」ボタンで結果を記録します。授業終了後に先生が一覧確認できます。

Aria
→
GAS（プロキシ）
→
Googleスプレッドシート
⚠️

保存ボタンはあらすじ生成後に有効になります。名前が空欄だと保存できません。

6

CanonへコピーしてノベルゲームにするのもOK（任意）

「コピー」ボタンで生成テキストをクリップボードにコピーし、Canon の「本文」欄に貼り付けることで、そのままノベルゲームに変換できます。

📜
Canon — Novel Editor

Canon の使い方

テキストを4ステップでノベルゲームJSON に変換するエディター

1

STEP 1 — 原稿を入力する

タイトル・ジャンルを入力し、本文テキストエリアに変換したい文章を貼り付けます。

  • 推奨文字数は 500〜2000文字
  • Ariaの生成結果・自作小説・読書感想文など何でもOK
  • 「サンプル」ボタンで羅生門・走れメロスなどを試せます
⚠️

2000文字を大きく超えると変換に失敗しやすくなります。長い場合は前半だけ貼るなど工夫してください。

2

「登場人物を抽出する」をタップ

AIがテキストから主要人物を最大4名まで自動抽出します。
抽出後は STEP 2 の「キャラクター確認・調整」が開きます。

  • 名前をクリックして変更できます
  • 色のサークルをクリックするとカラーピッカーが開きます（セリフの名前色になります）
  • 立ち位置（左／右）でキャラが画面のどちら側に表示されるか決まります
  • 「＋ キャラクターを追加」でキャラを増やせます
3

STEP 3 — 背景・BGMを選ぶ

STEP 2 が完了すると STEP 3 が開きます。チップをクリックして、物語に合う背景とBGMを選びます（複数選択OK）。

  • 背景：教室・廃校・屋上・森・宇宙船など 14 種類
  • BGM：不気味・切ない・コミカル・緊張など 9 種類
  • 選んだ候補をAIがシーンに合わせて割り当てます

選んだら「🎼 ノベルJSONに変換する」をタップ。30秒〜1分ほどかかることがあります。

4

STEP 4 — JSONをダウンロードする

変換結果のJSONプレビューが表示されます。

  • 「💾 .json をダウンロード」でファイルを保存
  • 「📋 コピー」でクリップボードにコピーも可能
💡

ダウンロードしたファイルを stories/ フォルダに入れて sonnet/stories.js に読み込ませるとSonnetで遊べます（詳細は「全体フロー」を参照）。

🎮
Sonnet — Novel Engine

Sonnet の使い方

サウンドノベルエンジン。作ったストーリーや組み込みサンプルをプレイできます

1

タイトル画面でストーリーを選ぶ

Sonnet を開くとタイトル画面が表示されます。収録されているストーリーの一覧からプレイしたいものを選んでクリック/タップします。

📖

初期収録は3本：廃校の約束（ホラー）・今日も放課後はカオスです（学園コメディ）・ZERO PROTOCOL（近未来シリアス）

2

クリック or タップで読み進める

画面をクリックまたはタップするとテキストが進みます。

  • 選択肢が出たらボタンをクリックして選んでください
  • 右上の ♪ ボタンでBGMのオン・オフを切り替えられます
  • ゲームエリア外のボタンで画面サイズプレビューを切り替えられます
3

エンディング後にタイトルへ戻る

エンディングに到達すると「タイトルに戻る」ボタンが表示されます。タップするとストーリー選択画面に戻って別の作品を選べます。

4

（上級）Canonで作ったJSONを追加する

Canonでダウンロードした .json ファイルを stories/ フォルダに置き、sonnet/stories.js の先頭の配列にファイルパスを追記することで自作ストーリーをプレイできます。

⚠️

ファイルをダブルクリックで直接開く場合（file:// プロトコル）は、ローカルファイルの読み込み制限があるため stories.js へのインライン埋め込みが必要です。HTTPサーバー経由で開けば通常のJSONロードが使えます。

Troubleshooting

よくあるトラブルと対処法

エラーが出たときはここを確認してください

症状 原因と対処
「Failed to fetch」「通信エラー」と表示される GASのデプロイURLが正しいか確認。または一時的なGASの不具合なので少し待ってから再試行してください。
「GEMINI_API_KEY が設定されていません」 GASのスクリプトプロパティに GEMINI_API_KEY が設定されていません。GASプロジェクトの「プロジェクトの設定」→「スクリプトプロパティ」から追加してください。
Gemini API エラー (HTTP 404) モデル名が間違っています。gas_proxy.gs の GEMINI_ENDPOINT が gemini-2.5-flash-lite になっているか確認し、再デプロイしてください。
保存ボタンを押しても何も起きない 名前欄が空欄か、あらすじが未生成の可能性があります。また SPREADSHEET_ID スクリプトプロパティが未設定の場合も失敗します。
Canon でキャラが1人も抽出されない テキストに台詞や登場人物の描写が少なすぎます。会話文を含む500文字以上のテキストを使ってください。
Canon でJSON変換が失敗する（エラー表示） テキストが長すぎるか、AIがJSONフォーマットを守れなかった可能性があります。テキストを短くして再試行するか、「↩ 最初から」でリセットしてください。
Sonnet でBGMが鳴らない ブラウザの自動再生ポリシーにより、最初の画面クリック前は音が出ません。ゲーム画面をクリックして操作を開始してください。右上の ♪ ボタンでオン/オフを確認することもできます。
タイトルに戻ると次のストーリーが始まらない ページをリロード（F5）してから再選択してください。
GASへのアクセスで認証画面が表示される デプロイ設定の「アクセスできるユーザー」が「全員」になっているか確認してください。「自分のみ」だとログイン画面が返ってきます。

Setup — 管理者向け

初期セットアップ手順

GAS・APIキー・スプレッドシートの設定（先生・管理者が行う作業）

S1

Google Apps Script プロジェクトを作成

script.google.com を開き、「新しいプロジェクト」を作成。aria/gas_proxy.gs の内容を全コピーして貼り付けます。

S2

スクリプトプロパティを設定

GASエディタの「プロジェクトの設定」→「スクリプトプロパティ」→「プロパティを追加」

  • GEMINI_API_KEY ：Gemini APIキー（Google AI Studio で取得）
  • SPREADSHEET_ID：保存先スプレッドシートのID（URLの /d/xxxxx/edit の xxxxx 部分）
S3

ウェブアプリとしてデプロイ

「デプロイ」→「新しいデプロイ」→「ウェブアプリ」を選択

  • 実行するユーザー：自分
  • アクセスできるユーザー：全員

デプロイ後のURLをコピーし、aria/index.html と canon/index.html の CONFIG.GAS_URL に貼り付けます。

⚠️

既存デプロイを更新する場合：「デプロイを管理」→「鉛筆アイコン」→「バージョン：新しいバージョン」を選択してデプロイしてください。URLは変わりません。

S4

疎通確認

ブラウザで以下のURLにアクセスし、JSONレスポンスが返ってくれば成功です。

GAS疎通確認URL（ブラウザで開く）
https://script.google.com/macros/s/[デプロイID]/exec
期待されるレスポンス
{"status":"ok","message":"GAS proxy is running"}
📌

現在設定済みのGAS URL：https://script.google.com/macros/s/AKfycbzdSAoFA7G9ByUdbQDqDIUcDTn4iCI-vhD37BSJFK2CMzO7awdY_kz1j3SZdjnW8eXv/exec

Multi Device — 複数端末セットアップ

複数の端末で使うには

リポジトリをクローンして同じ GAS URL を設定するだけ。スプシが共有ハブになります

1

Git でリポジトリをクローン

新しい端末でターミナル（または Git Bash）を開き、以下のコマンドを実行します。

ターミナル
git clone https://github.com/RyutaAwashima/AI-------20260511.git
cd AI-------20260511

Git が入っていない場合は git-scm.com からインストールしてください。または ZIP でダウンロードして展開しても OK です。

2

GAS URL を 3 ファイルに設定

以下の 3 箇所を開いて、GAS デプロイ URL を貼り付けます（同じ URL を 3 ファイルすべてに設定してください）。

ファイル設定箇所
aria/index.html 約554行目：CONFIG.GAS_URL = '...'
canon/index.html 約443行目：CONFIG.GAS_URL = '...'
sonnet/engine.js 約8行目：CLOUD_CONFIG.GAS_URL = '...'
💡

全員が同じ GAS URL を使えば、誰が作ったストーリーも同じスプシに集まります。授業・ワークショップで複数台同時使用する場合にも便利です。

3

guide.html をブラウザで開くだけ

サーバーは不要です。guide.html をダブルクリックするか、ブラウザに file:// パスをドラッグ＆ドロップすれば起動できます。起動ボタンから各アプリに移動してください。

🔑

リポジトリがプライベートの場合、クローンには GitHub アカウントが必要です。現在は作者 (RyutaAwashima) 以外にアクセス権がありません。共有したい相手を GitHub の「Settings → Collaborators」から招待してください。

Roadmap — 将来計画

ウェブサービス化ロードマップ

現在のローカル運用から、いつかの公開サービスまで

Phase 0
🏠
現在

ローカル PC + 個人 GAS（クローズドβ）

各自の PC にリポジトリをクローンし、自分の GAS & スプシで動かす形態。インターネット接続は Gemini API と GAS の呼び出しのみ。

  • コスト：ほぼ無料（Gemini 無料枠 + Google Apps Script 無料枠）
  • 制限：1 つの GAS アカウントの枠内に収まる規模のみ
Phase 1
🌐
近期

LAN / Python HTTP サーバー共有

授業・ワークショップなど同一ネットワーク内で複数人が同時利用できる構成。
講師 PC で python -m http.server 8080 を起動するだけで参加者がブラウザからアクセス可能になります。

  • 追加コスト：なし（既存環境のまま）
  • 必要作業：URL 配布のみ（設定変更不要）
Phase 2
📦
中期

GitHub Pages で静的公開

フロントエンド（HTML/CSS/JS）を GitHub Pages に公開して固定 URL を持たせる。GAS & スプシは引き続き個人アカウントのものを使用。

  • URL が固定されリンク共有が簡単に
  • リポジトリを Public にする必要あり（または GitHub Pro のプライベートページ）
  • マルチユーザー対応には別途バックエンドが必要
Phase 3
⚡
将来

バックエンドサービス化（マルチユーザー対応）

GAS を廃止し、Vercel Functions / Cloud Run などでバックエンド API を構築。Firebase または Supabase でユーザー認証とデータ管理を行います。

  • 複数ユーザーが独立したスプシ（またはDB）を持てる
  • Gemini API キーをサーバー側で管理しクライアントに露出させない
  • コスト：月数百〜数千円（規模次第）
Phase 4
🚀
SaaS

認証・マルチテナント・有料プラン（SaaS）

ユーザーアカウント登録、作品の公開/非公開設定、課金プランなどを備えた本格的なウェブサービス化。

  • フリー枠：月 X 作品まで無料生成
  • プロプラン：生成回数無制限・カスタムキャラ・BGM アップロード
  • 技術：Next.js / Stripe / Supabase Auth など
